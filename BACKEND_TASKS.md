# Backend — выполненные задачи

Проект: **Spotify Clone** (FastAPI + PostgreSQL + Docker)

---

## `app/routes/auth.py` — Аутентификация

- Регистрация пользователя (email + bcrypt-хеш пароля)
- Вход в систему с выдачей JWT access + refresh токенов
- Проверка токена на каждом защищённом эндпоинте
- Алгоритм HS256, настраиваемое время жизни токена

---

## `app/routes/songs.py` — Треки

- GET список всех треков с пагинацией
- GET featured треки (для главной страницы)
- GET trending треки
- POST создание трека (только для админа)
- PUT редактирование трека
- DELETE удаление трека
- Исправление: добавлен `@router.post("")` — устранён 307-редирект в Docker
- Исправление: трек автоматически наследует `image_url` альбома при создании

---

## `app/routes/albums.py` — Альбомы

- GET список всех альбомов
- GET альбом по ID с вложенными треками
- POST создание альбома (только для админа)
- PUT редактирование альбома
- DELETE удаление альбома

---

## `app/routes/search.py` — Поиск

- GET поиск треков, альбомов и артистов по строке запроса
- Поиск треков по полям: `title`, `artist`, `album_name`
- Поиск альбомов по `title` и `artist`
- Исправление: добавлен `@router.get("")` — устранён 307-редирект в Docker/Nginx
- Расширение: добавлен поиск по полю `album_name` (ранее отсутствовал)

---

## `app/routes/upload.py` — Загрузка файлов

- POST загрузка изображения (jpg, png, webp) → сохранение в `/media/images/`
- POST загрузка аудиофайла (mp3) → сохранение в `/media/songs/`
- Генерация уникального имени файла через UUID (исключает конфликты имён)

---

## `app/routes/playlists.py` — Плейлисты

- GET плейлисты текущего пользователя (`/playlists/me`)
- POST создание плейлиста
- PUT редактирование плейлиста
- DELETE удаление плейлиста
- POST добавление трека в плейлист
- DELETE удаление трека из плейлиста

---

## `app/routes/seed.py` — Тестовые данные

- POST заполнение БД тестовыми пользователями, альбомами и треками
- Поддержка параметра `?force=true` — пересоздание данных с нуля

---

## `app/routes/users.py` — Профили пользователей

- GET профиль текущего пользователя
- PUT обновление профиля (имя, аватар, bio)

---

## `app/routes/player.py` — История воспроизведения

- POST запись факта прослушивания трека в `listening_history`

---

## `app/routes/recommendations.py` — Рекомендации

- GET рекомендованные треки на основе истории прослушиваний пользователя

---

## `app/routes/websocket.py` — WebSocket чат

- WebSocket-соединение для обмена сообщениями в реальном времени
- Трансляция сообщений между всеми подключёнными пользователями

---

## Nginx (`frontend/nginx.conf`)

- Увеличен `client_max_body_size` до 100MB — исправлена ошибка 413 при загрузке файлов
- Настроено проксирование `/api/*` и `/media/*` на backend (порт 8000)
- Убран MP3 из правил кэширования статики — аудиофайлы теперь проксируются корректно
- Настроен `try_files $uri /index.html` — поддержка React SPA-маршрутизации

## Исправленные баги (Backend)

| Баг | Причина | Решение |
|-----|---------|---------|
| Поиск возвращал пустой результат | 307-редирект FastAPI + Nginx проксировал внутренний Docker-IP | Добавлен `@router.get("")` к search-эндпоинту |
| Трек не создавался через админ-панель | `POST /songs` без слэша вызывал 307-редирект | Добавлен `@router.post("")` к songs-эндпоинту |
| Аудио не воспроизводилось | Nginx кэшировал `*.mp3` из статики вместо проксирования | Убран mp3 из regex-правила кэша в nginx.conf |
| 413 Payload Too Large при загрузке файлов | Лимит Nginx по умолчанию 1MB | `client_max_body_size 100M` в nginx.conf |
| Поиск не находил треки по названию альбома | Фильтр охватывал только `title` и `artist` | Добавлен `Track.album_name.ilike(...)` в OR-фильтр |
| Трек создавался без обложки | Backend не подставлял `image_url` альбома | Backend автоматически копирует `image_url` альбома в трек |
