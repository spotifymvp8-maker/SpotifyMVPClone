# Architecture Request Flow

**Изображения:** `architecture-request-flow.png` (оригинал) | `architecture-request-flow-detailed.png` (подробная версия)

---

## Что изображено

Схема маршрутизации HTTP-запросов от пользователя до сервисов приложения.

- **Пользователь/браузер** — источник запросов
- **Nginx** — точка входа (порт 3000 или 80)
- **React SPA** — статические файлы приложения
- **FastAPI** — REST API и медиафайлы
- **PostgreSQL** — база данных

---

## Как это работает

1. **Пользователь** отправляет запрос на `http://localhost:3000` (или другой домен).
2. **Nginx** анализирует путь запроса и направляет его:

  | Путь       | Куда идёт           | Что возвращается      |
  | ---------- | ------------------- | --------------------- |
  | `/`        | React SPA           | `index.html`, JS, CSS |
  | `/api/`*   | Backend (порт 8000) | JSON-ответы API       |
  | `/media/*` | Backend             | Аудиофайлы, обложки   |
  | `/ws/*`    | Backend             | WebSocket (чат)       |

3. **Backend** для `/api/`* и `/media/*`:
  - Выполняет бизнес-логику
  - Обращается к PostgreSQL
  - Возвращает JSON или файл
4. **PostgreSQL** хранит данные; backend читает и записывает через SQLAlchemy.
5. В итоге пользователь получает либо HTML/JS (страница), либо JSON (данные), либо медиафайл (аудио).
