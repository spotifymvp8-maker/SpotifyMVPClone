# Работа с Jamendo API

Диаграмма описывает интеграцию приложения Spotify Clone с внешним API музыкального сервиса Jamendo для поиска и импорта бесплатных треков под лицензиями Creative Commons.

---

## Что изображено на диаграмме

### Участники

| Компонент | Роль |
|-----------|------|
| **Admin Panel** | Интерфейс администратора: поле поиска, жанровые теги, список треков, кнопки «Прослушать» и «Добавить» |
| **Frontend (React)** | Компонент `JamendoImportModal` — модальное окно импорта из Jamendo |
| **Backend (FastAPI)** | Два эндпоинта: `GET /api/jamendo/search` и `POST /api/jamendo/import` |
| **Jamendo API** | Внешний сервис `api.jamendo.com` — поиск треков по запросу, артисту, жанру |
| **Jamendo CDN** | `prod-1.storage.jamendo.com` — хранение аудиофайлов, прямой стриминг в браузер |
| **PostgreSQL** | Локальная БД: таблица `tracks` с `file_url` (содержит `trackid` из Jamendo) |

---

## Поток данных

### 1. Поиск треков

1. Администратор вводит запрос (имя артиста или название трека) или выбирает жанр.
2. Frontend отправляет `GET /api/jamendo/search?q=...&tags=...&limit=30`.
3. Backend:
   - запрашивает `PostgreSQL` — получает список уже импортированных `trackid`;
   - отправляет запрос `GET https://api.jamendo.com/v3.0/tracks/` с `client_id`, `search`, `artist_name`, `fuzzytags` и т.п.;
   - получает список треков с `audio_url`, `image_url`, `title`, `artist`;
   - исключает треки, чьи `trackid` уже есть в БД;
   - возвращает отфильтрованный список в Frontend.
4. Frontend отображает список треков в модале.

### 2. Предпрослушивание

1. Администратор нажимает ▶ на треке.
2. Браузер воспроизводит аудио **напрямую** с Jamendo CDN по `audio_url` (без участия нашего сервера).
3. Трек не скачивается на сервер — используется только потоковая передача.

### 3. Импорт трека

1. Администратор нажимает «Добавить».
2. Frontend отправляет `POST /api/jamendo/import` с телом: `jamendo_id`, `title`, `artist`, `album_name`, `duration`, `audio_url`, `image_url`.
3. Backend:
   - проверяет дубль по `trackid` в `file_url` (если уже есть — возвращает `already_exists`);
   - скачивает обложку по `image_url` и сохраняет в `/media/images/`;
   - создаёт запись в таблице `tracks` с `file_url = audio_url` (прямая ссылка на Jamendo) и `image_url = /media/images/xxx.jpg` (локальная копия);
   - возвращает `201 Created` с данными трека.
4. Frontend обновляет список и помечает трек как добавленный.

## Ключевые технические решения

| Решение | Причина |
|---------|---------|
| **Аудио не скачивается** | Экономия места на диске; быстрый импорт; стриминг напрямую с Jamendo CDN |
| **Обложки скачиваются** | Надёжность отображения; независимость от внешнего CDN; единый путь `/media/images/` |
| **Сравнение по trackid** | URL Jamendo содержит временный токен `from=...`, который меняется при каждом запросе. Стабильный идентификатор — `trackid=XXXX` в URL |
| **Фильтрация уже импортированных** | В модале показываются только новые треки; нет дублей в библиотеке |
| **client_id** | Jamendo требует регистрацию на devportal.jamendo.com; бесплатный ключ для non-commercial использования |
