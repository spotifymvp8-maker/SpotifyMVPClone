## Задача 1: Shuffle — воспроизведение в случайном порядке

### Требования

- Кнопка Shuffle рядом с элементами управления плеером
- При включении: треки воспроизводятся в случайном порядке **без повторений** до тех пор, пока не сыграют все
- При выключении: возврат к обычной очереди
- Визуальная индикация состояния (подсветка кнопки)
- Работает на страницах Playlist и Album

### Реализация

**Алгоритм Fisher-Yates** — используется для перемешивания массива треков:

```typescript
// usePlayerStore.ts
function fisherYates<T>(arr: T[]): T[] {
    const a = [...arr];
    for (let i = a.length - 1; i > 0; i--) {
        const j = Math.floor(Math.random() * (i + 1));
        [a[i], a[j]] = [a[j], a[i]];
    }
    return a;
}
```

**Почему Fisher-Yates?**
В отличие от `array.sort(() => Math.random() - 0.5)` (статистически нечестный), алгоритм Fisher-Yates гарантирует равномерное распределение — каждая перестановка массива равновероятна.

**State в `usePlayerStore`:**

```typescript
isShuffled: boolean          // включён ли shuffle
shuffledQueue: Song[]        // перемешанная очередь
shuffledIndex: number        // текущая позиция в перемешанной очереди
```

**Логика `playNext()` с shuffle:**

```typescript
playNext: () => {
    const { isShuffled, shuffledQueue, shuffledIndex, repeatMode } = get();

    if (isShuffled) {
        if (shuffledIndex < shuffledQueue.length - 1) {
            // Переходим к следующему треку в перемешанной очереди
            const nextIndex = shuffledIndex + 1;
            set({ shuffledIndex: nextIndex, currentSong: shuffledQueue[nextIndex] });
        } else if (repeatMode === "all") {
            // Все треки сыграли — перемешиваем заново и начинаем
            const newShuffled = fisherYates(get().queue);
            set({ shuffledQueue: newShuffled, shuffledIndex: 0, currentSong: newShuffled[0] });
        }
        // repeatMode === "off" — ничего не делаем (очередь кончилась)
    } else {
        // Обычная линейная очередь
        ...
    }
}
```

**UI в `PlaybackControls.tsx`:**

```tsx
<button
    onClick={toggleShuffle}
    className={`transition-colors ${isShuffled ? "text-green-400" : "text-zinc-400 hover:text-white"}`}
>
    <Shuffle className="h-4 w-4" />
    {isShuffled && (
        <span className="absolute bottom-0 left-1/2 -translate-x-1/2 w-1 h-1 rounded-full bg-green-400" />
    )}
</button>
```

### Результат

- Треки воспроизводятся в случайном порядке без повторений
- После того как все треки в очереди сыграли — очередь перемешивается заново (при `Repeat All`)
- При отключении Shuffle — возврат к текущей позиции в обычной очереди
- Кнопка подсвечивается зелёным + зелёная точка-индикатор снизу

## Задача 2: Repeat — повторное воспроизведение

### Требования

- Три режима: **Off** → **Repeat All** → **Repeat One**
- Визуально различимы: иконка `1` при Repeat One
- Работает без багов как для одного трека, так и для плейлиста/альбома

### Реализация

**State в `usePlayerStore`:**

```typescript
repeatMode: "off" | "all" | "one"
```

**Переключение по циклу:**

```typescript
toggleRepeat: () => {
    set(s => ({
        repeatMode: s.repeatMode === "off" ? "all"
                  : s.repeatMode === "all" ? "one"
                  : "off"
    }));
}
```

**Обработка в `AudioPlayer.tsx`:**

```typescript
const handleEnded = () => {
    if (repeatMode === "one") {
        // Repeat One: сбрасываем время и продолжаем
        if (audioRef.current) {
            audioRef.current.currentTime = 0;
            audioRef.current.play();
        }
    } else {
        // Off / All: переходим к следующему треку (логика в store)
        playNext();
    }
};
```

**Разделение ответственности:**
- `AudioPlayer.tsx` — знает про HTML-элемент `<audio>`, обрабатывает Repeat One
- `usePlayerStore.playNext()` — знает про очередь, обрабатывает Off и All
- `PlaybackControls.tsx` — только отображает кнопку и передаёт действия

**UI — отображение трёх состояний:**

```tsx
<button onClick={toggleRepeat}>
    {repeatMode === "one" ? (
        <div className="relative">
            <Repeat className="h-4 w-4 text-green-400" />
            <span className="absolute -top-1 -right-1 text-[8px] font-bold text-green-400">1</span>
        </div>
    ) : (
        <Repeat className={`h-4 w-4 ${repeatMode === "all" ? "text-green-400" : "text-zinc-400"}`} />
    )}
</button>
```

### Результат

- **Off**: воспроизведение останавливается после последнего трека
- **Repeat All**: по окончании последнего трека плейлист начинается заново
- **Repeat One**: текущий трек повторяется бесконечно
- Состояние видно по цвету кнопки (зелёный = активен) и иконке `1`

## Задача 5: Интеграция с Jamendo API

### Контекст

Jamendo — платформа с более чем 500 000 бесплатных треков под лицензиями Creative Commons. Интеграция позволяет администратору находить и добавлять музыку без необходимости самостоятельно загружать файлы.

### Архитектура решения

```
Admin Panel  →  /api/jamendo/search  →  Jamendo API (api.jamendo.com)
                                         ↓
Admin Panel  →  /api/jamendo/import  →  Скачать обложку  →  Сохранить в media/images/
                                         ↓
                                        Создать Track в PostgreSQL
                                         (file_url = прямой стриминг с Jamendo CDN)
```

**Ключевое решение:** аудиофайлы **не скачиваются** на сервер — браузер стримит их напрямую с серверов Jamendo по прямой ссылке. Это экономит место на диске и ускоряет импорт.

### Бэкенд — `app/routes/jamendo.py`

#### GET /api/jamendo/search

Стратегия поиска в два шага:

```python
# Шаг 1: поиск по имени артиста (точный)
tracks = _fetch_jamendo_tracks({"artist_name": q, ...})

if not tracks:
    # Шаг 2: фолбэк — свободный текстовый поиск (трек, альбом, теги)
    tracks = _fetch_jamendo_tracks({"search": q, ...})
```

Если запрос пустой — возвращаются популярные треки (`order=popularity_month`).

#### POST /api/jamendo/import

```python
def import_jamendo_track(body: JamendoImportRequest, ...):
    # Проверка на дубль по audio_url (уникальный идентификатор трека)
    existing = db.query(Track).filter(Track.file_url == body.audio_url).first()
    if existing:
        return {"already_exists": True, ...}

    # Скачиваем обложку и сохраняем локально
    local_image_url = _download_image(body.image_url)

    track = Track(
        title=body.title,
        artist=body.artist,
        album_name=body.album_name or body.title,
        duration=body.duration,
        file_url=body.audio_url,            # прямая ссылка Jamendo
        image_url=local_image_url or body.image_url,  # локальная копия обложки
    )
    db.add(track)
    db.commit()
```

#### Скачивание обложки

```python
def _download_image(url: str) -> str | None:
    """Скачать обложку с Jamendo CDN и сохранить в /media/images/."""
    try:
        IMAGES_DIR.mkdir(parents=True, exist_ok=True)
        with httpx.Client(timeout=10.0) as client:
            resp = client.get(url, follow_redirects=True)
            resp.raise_for_status()
        ext = ".png" if "png" in resp.headers.get("content-type","") else ".jpg"
        filename = f"{uuid4().hex}{ext}"
        (IMAGES_DIR / filename).write_bytes(resp.content)
        return f"/media/images/{filename}"
    except Exception:
        return None   # fallback на оригинальный URL Jamendo
```

## Задача 6: Исправление ошибок импорта треков

### Ошибка 1: `album_name NOT NULL violation`

**Симптом:** при нажатии «Добавить» — тост «Ошибка импорта трека».

**Лог бэкенда:**
```
sqlalchemy.exc.IntegrityError: null value in column "album_name"
of relation "tracks" violates not-null constraint
```

**Причина:** `JamendoImportRequest` не содержал поле `album_name`, поэтому `Track` создавался с `album_name=None`.

**Исправление:**

```python
# jamendo.py — схема запроса
class JamendoImportRequest(BaseModel):
    jamendo_id: str
    title: str
    artist: str
    album_name: str = ""   # ← добавлено
    duration: int
    audio_url: str
    image_url: str

# При создании трека
track = Track(
    album_name=body.album_name or body.title,  # ← fallback на название трека
    ...
)
```

```typescript
// JamendoImportModal.tsx — добавлено в запрос
const res = await axiosInstance.post("/jamendo/import", {
    ...
    album_name: track.album_name,   // ← добавлено
});
```

## Задача 7: Исправление отображения на главной странице

### Проблема 1: Новые треки не появлялись на главной

**Причина:** `featured`, `made-for-you`, `trending` использовали `db.query(Track).limit(10).all()` без сортировки. PostgreSQL возвращал первые 10 строк в порядке хранения на диске (обычно — самые старые). Недавно импортированные треки не попадали в выборку.

**Исправление в `app/routes/songs.py`:**

```python
# БЫЛО: первые 10 треков без порядка
tracks = db.query(Track).limit(10).all()

# СТАЛО: последние 50 треков → случайная выборка из них
tracks = db.query(Track).order_by(Track.created_at.desc()).limit(50).all()
return random.sample(tracks, min(len(tracks), 6))
```

Теперь недавно добавленные треки (включая импортированные из Jamendo) попадают в случайную выборку для главной страницы.

### Проблема 2: Картинки медиафайлов возвращали 404

**Причина:** конфликт правил в `nginx.conf`.

В nginx правила обрабатываются в порядке приоритета:
1. `=` — точное совпадение (наивысший)
2. `^~` — prefix без regex (блокирует дальнейший поиск)
3. `~` / `~*` — regex (в порядке объявления)
4. Prefix без модификатора (низший)

**Было:**
```nginx
location /media { proxy_pass http://backend:8000; }          # prefix (низший)
location ~* \.(jpg|png|webp|...)$ { expires 1y; }            # regex (выше!)
```

Запрос `/media/images/abc.jpg` совпадал с regex `~* \.jpg$` → nginx пытался отдать файл из своей статики `/usr/share/nginx/html/media/images/abc.jpg` → файла нет → **404**.

**Исправление:**
```nginx
location ^~ /media {    # ^~ блокирует проверку regex
    proxy_pass http://backend:8000;
    ...
}
```

Теперь любой URL начинающийся с `/media` гарантированно проксируется на бэкенд.

### Ключевые технические решения сессии

1. **Fisher-Yates для Shuffle** — математически честный алгоритм перемешивания, гарантирующий равновероятность всех перестановок
2. **Разделение ответственности Repeat** — AudioPlayer отвечает за Repeat One (работа с HTML `<audio>`), store — за Repeat All (работа с очередью)
3. **Jamendo: стриминг без скачивания** — аудио воспроизводится напрямую с CDN Jamendo, изображения сохраняются локально
4. **Nginx `^~ /media`** — правильный приоритет location-блоков предотвращает перехват media-запросов regex-правилом статики
5. **`ORDER BY created_at DESC`** — гарантирует, что новые треки попадают в выборку для главной страницы
