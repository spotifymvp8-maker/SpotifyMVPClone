## Что изображено

Схема высокоуровневой архитектуры проекта Spotify Clone. Все компоненты показаны как работающие в Docker-контейнерах.

- **Браузер/клиент** — пользователь открывает приложение в браузере
- **Nginx** — обратный прокси, принимает запросы на портах 80 (HTTP) и 443 (HTTPS)
- **Frontend** — React SPA (собранный Vite), статические файлы (HTML, JS, CSS)
- **Backend** — FastAPI + Uvicorn, REST API, Swagger-документация
- **PostgreSQL 16** — реляционная база данных
- **Media volume** — хранилище загруженных аудиофайлов и изображений

---

## Как это работает

1. **Пользователь** обращается к приложению (например, `http://localhost:3000`).
2. **Nginx** принимает запрос и маршрутизирует его:
  - `/` и статика → отдаёт файлы React-приложения
  - `/api/`* → проксирует на FastAPI (порт 8000)
  - `/media/*` → проксирует на FastAPI (файлы из папки media)
  - `/ws/*` → проксирует WebSocket-соединения
3. **Backend** обрабатывает API-запросы, обращается к PostgreSQL за данными и к media volume за файлами.
4. **PostgreSQL** хранит пользователей, альбомы, треки, плейлисты, историю прослушиваний.
5. Все сервисы запускаются одной командой `docker compose up -d --build`.
